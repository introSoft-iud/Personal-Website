Guía de Configuración de Git y GitHub

En este tutorial, aprenderás cómo instalar y configurar Git, así como vincularlo con GitHub para gestionar tus repositorios de manera eficiente.

Iniciemos describiendo qué es Git y su flujo de trabajo:

¿Qué es Git?

Git es un sistema de control de versiones que te permite gestionar y rastrear los cambios en tus proyectos de software. Con Git, puedes trabajar en diferentes versiones de tu código al mismo tiempo, colaborar con otros desarrolladores y volver a versiones anteriores si es necesario.

El flujo de trabajo en Git

El flujo de trabajo en Git sigue tres pasos principales para gestionar los cambios en tus archivos:

1. Área de preparación (staging)

Cuando realizas cambios en los archivos de tu proyecto, primero debes agregarlos al área de preparación o "staging". Esto indica que los archivos están listos para ser confirmados (commit). Para agregar un archivo al área de preparación

2. Confirmar los cambios (commit)

Una vez que los archivos están en el área de preparación, el siguiente paso es confirmarlos, lo que significa que se guardan de manera permanente en el historial del proyecto.

3. Enviar los cambios (push)

Después de confirmar los cambios localmente, puedes enviarlos al repositorio remoto (como GitHub) para que se guarden en línea y puedan ser compartidos con otros desarrolladores.

Ahora si podemos iniciar el paso a paso desde la instlación y configuración de GIT hasta hasta su vinculación con el repositorio remoto, que en este caso el que vamos a utilizar es GITHUB.

Paso 1: Instalación de Git

Primero, necesitamos instalar Git. Dependiendo del sistema operativo, los pasos varían. Visita el sitio oficial de Git para descargar la versión adecuada.

Como en mi caso el sistema operativo que estoy utilizando es Linux los pasos que se deben realizar desde la terminal son:

1. Verificar la instalación de git

git --version

Como aparece la versión de git, significa que ya está instalada,en el caso de que no apareciera la versión de git lo que se debe hacer para instalarlo es:

1.1 Actualizar la lista de paquetes

sudo apt update

1.2 Instalar git

sudo apt install git

1.3 Verificar la instlación

git --version

Paso 2: Configuración inicial de Git

Una vez instalado, abre la terminal y configura tu nombre, correo electrónico y verifica la configuración con los siguientes comandos:

git config --global user.name "Tu Nombre"
git config --global user.email "tuemail@dominio.com"
git config --list

Paso 3: Inicializar un repositorio de git

Cuando ya hayas realizado las configuraciones iniciales puedes inicializar un repositorio de git siguiendo los siguientes pasos:

1.1 Si aún no tienes una carpeta para tu proyecto la puedes crear desde la terminal desde la ruta en donde quiere ubicarlo

mkdir nombre-proyecto
cd nombre-proyecto

1.2 Inicializa el repositorio utilizando el siguiente comando

git init

1.3 Agrega los archivos al repositorio y confirma los cambios

Con este comando se agregan todos los archivos al area de staging (area de preparación):

git add .

Para guardar los cambios que agregaste al área de preparación, haz un commit. Un commit es como una "foto" de tu proyecto en un momento específico: 

git commit -m "Mensaje descriptivo de los cambios"

Paso 4: Configurar Llaves SSH

Para evitar tener que ingresar tu nombre de usuario y contraseña cada vez que trabajas con GitHub, puedes configurar una llave SSH. Para hacerlo, sigue estos pasos:

Generar una nueva llave SSH

En la terminal, ejecuta el siguiente comando para generar una nueva llave SSH:

ssh-keygen -t ed25519 -C "tuemail@dominio.com"

Luego, presiona Enter para aceptar la ubicación por defecto y Enter nuevamente para dejar en blanco la contraseña opcional.

Agregar la llave SSH al agente SSH

Para agregar tu nueva llave SSH al agente, usa los siguientes comandos:

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

Agregar la llave SSH a GitHub

Copia el contenido de tu llave SSH con el siguiente comando:

cat ~/.ssh/id_ed25519.pub

Ve a GitHub, en tu perfil, selecciona Settings > SSH and GPG keys, y haz clic en New SSH key. Pega el contenido de la llave copiada y guárdala.

Configurar git para usar SSH

En mi caso como ya habia realizado la clonación del repositorio usando HTTPS y no SSH, lo que se debe hacer es cambiar el URL del repositorio a SSH con el siguiente comando: 

git remote set-url origin git@github.com:usuario/repo.git

Paso 3: Crear un repositorio en GitHub

Si aún no creaste el repositorio en Github. Ve a tu cuenta de GitHub y selecciona New Repository. Dale un nombre a tu repositorio y elige si deseas que sea público o privado.

Paso 4: Hacer push de los cambios

Como ya hiciste un commit de los cambios que realizaste, sigue los siguientes pasos:

1. Cambia el nombre a la rama master

git branch -M main

2. Añade el link del repositorio remoto configurado con llaves ssh

git remote add origin git@github.com:yennivanessa97/Personal-Website-example.git

3. Empuja los cambio con el comando push

git push origin main

Paso 5: Crear ramas usando GitHub Flow

El flujo de trabajo en GitHub se llama GitHub Flow, y está basado en el uso de ramas. Siguiendo este flujo, puedes desarrollar nuevas características o arreglar errores sin afectar el código principal. Los pasos básicos son:

Crear una nueva rama

Para crear una nueva rama, usa el siguiente comando:

git checkout -b nombre-de-la-rama

Trabajar en la rama

Realiza tus cambios en la nueva rama y luego haz commit como normalmente lo harías.
